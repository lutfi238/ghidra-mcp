# v4.0.0 Monolith Refactor Plan

**Status**: Complete
**Target**: v4.0.0
**Started**: 2026-02-27
**Completed**: 2026-02-28

## Problem

`GhidraMCPPlugin.java` is 16,945 lines containing 149 HTTP endpoints and 257 private helper methods. `HeadlessEndpointHandler.java` duplicates ~6,000 lines of the same business logic for headless mode. This makes the codebase difficult to navigate, test, and extend.

## Solution

Extract shared business logic into 10 service classes under `com.xebyte.core`. The plugin becomes a ~1,200-line thin orchestrator (server lifecycle + endpoint wiring). The headless handler shrinks to ~300 lines. No HTTP paths, parameter names, or response formats change.

## Key Design Decisions

1. **Instance-based services with constructor injection** — Services receive `ProgramProvider` + `ThreadingStrategy` via constructor. Reuses existing abstractions that already work in GUI (`SwingThreadingStrategy`) and headless (`DirectThreadingStrategy`) modes.

2. **Keep `server.createContext()` pattern** — The existing endpoint registration pattern is proven and grep-friendly. No new routing framework needed.

3. **Services return JSON strings** — Same as today. No `Response` sealed interface or serialization library changes.

4. **GUI-only endpoints stay in plugin** — `getCurrentAddress()` and `getCurrentFunction()` use `CodeViewerService` which has no headless equivalent.

## Threading Migration Pattern

The key migration for write operations:

```java
// BEFORE (manual SwingUtilities + transaction in GhidraMCPPlugin.java):
SwingUtilities.invokeAndWait(() -> {
    int tx = program.startTransaction("Rename function");
    boolean success = false;
    try {
        function.setName(newName, SourceType.USER_DEFINED);
        success = true;
    } finally {
        program.endTransaction(tx, success);
    }
});

// AFTER (delegated to ThreadingStrategy in service):
return threadingStrategy.executeWrite(program, "Rename function", () -> {
    function.setName(newName, SourceType.USER_DEFINED);
    return "{\"status\": \"success\"}";
});
```

`SwingThreadingStrategy.executeWrite()` handles EDT dispatch, transaction start/commit/rollback, and event flushing automatically.

## What Does NOT Change

- HTTP endpoint paths and parameters
- JSON response formats
- MCP bridge (`bridge_mcp_ghidra.py`) — all 180 tools unchanged
- `endpoints.json` test catalog
- `ProgramProvider`/`ThreadingStrategy` interfaces
- `BinaryComparisonService` (already extracted)
- Static singleton HTTP server pattern (multi-window support)

## Not Included (Future Work)

- `Response` sealed interface for type-safe returns (v5.0)
- JSON serialization library (Gson/Jackson) (v5.0)
- Declarative endpoint routing (unnecessary for current needs)
